New Project Ltd. – Time Card Management System

Front-end mockup for the TimeCard database (no backend / database connected).

Modules

Employees Projects Clients Time Cards Work & Expense Codes Payments

Employees

Manage employee records and billing rates.

Add / Edit Employee

Attributes: Employee ID, First Name, Last Name, Title, Phone, Billing Rate

EmployeeID is auto-generated. Title is limited to the listed values. BillingRate must be greater than 0. FirstName and Phone are mandatory and Phone must match 11-111-1111-111-111.

Projects

Track projects, clients, estimates and assigned managers.

Add / Edit Project

Attributes: Project ID, Project Name, Project Description, Client ID, Billing Estimate, Employee ID (manager), Start Date, End Date

ProjectID is auto-generated. ProjectName, StartDate and EndDate are mandatory. BillingEstimate must be greater than 1000. EndDate should be greater than StartDate. ClientID is a foreign key from CustomerDetails.Clients.

Clients

Maintain client contact details used for projects and billing.

Add / Edit Client

Attributes: Client ID, Company Name, Address, City, State, Zip, Country, Contact Person, Phone

ClientID is auto-generated. CompanyName, ContactPerson, Address, City, State, Zip, Country and Phone are mandatory. Phone must match 11-111-1111-111-111.

Time Cards

Capture hours and expenses for billable work.

Time Card Header

Attributes: Time Card ID, Employee ID, Project ID, Date Issued, Days Worked

TimeCardID is auto-generated. EmployeeID and ProjectID are foreign keys. DateIssued should be greater than the current date and the project start date. DaysWorked must be greater than 0.

Time Card Hours (Detail)

Attributes: Time Card Detail ID, Time Card ID, Date Worked, Project ID, Work Description, Billable Hours (> 0), Total Cost (auto), Work Code ID

Billable hours must be greater than 0. TotalCost is calculated as Billable Hours * BillingRate in the database.

Time Card Expenses

Attributes: Time Card Expense ID, Time Card ID, Expense Date, Project ID, Expense Description, Expense Amount (> 0), Expense ID (FK)

TimeCardExpenseID is auto-generated. ExpenseDate cannot be NULL and should be before the project end date. ExpenseAmount must be greater than 0. ExpenseID is a foreign key from ExpenseDetails.

Payments

Record client payments against projects.

Payment Entry

Attributes: Payment ID, Project ID, Payment Amount (> 0), Payment Date, Credit Card Number, Card Holder's Name, Credit Card Expiry Date, Payment Method ID, Payment Due

PaymentID is auto-generated. PaymentAmount must be greater than 0. PaymentDate should be greater than the project end date. If a credit card is not used, card fields may be left blank. If a credit card is used, card fields must be filled and Expiry should be after PaymentDate. PaymentDue should not be greater than PaymentAmount.

Payment Methods

Attributes: Payment Method ID, Description (NOT NULL)

PaymentMethodID is auto-generated. Description cannot be blank.
